# La Línea de Comandos: AWS CLI

La **AWS CLI** (Command Line Interface, interfaz de línea de comandos) es una herramienta oficial
de Amazon que permite interactuar con todos los servicios de AWS directamente desde la terminal,
sin necesidad de abrir el navegador ni la consola web. Con un solo comando puedes lanzar un servidor,
crear un bucket, gestionar permisos o desplegar una aplicación completa.

Si la consola web es el panel de control visual, la CLI es el teclado directo a la API de AWS.
Ambas hacen exactamente lo mismo por debajo — la consola web también llama a la misma API — pero
la CLI tiene ventajas que la hacen indispensable en la práctica profesional:

- **Automatización** — puedes encadenar comandos en scripts que se ejecutan sin intervención humana.
- **Reproducibilidad** — un script de CLI documenta exactamente qué se hizo y puede repetirse sin errores de clic.
- **Velocidad** — para operaciones repetitivas es órdenes de magnitud más rápido que navegar la consola.
- **Integración con CI/CD** — pipelines de despliegue automatizado dependen de la CLI para interactuar con AWS.

::: info Documentación oficial de la AWS CLI
- Página principal: [aws.amazon.com/cli](https://aws.amazon.com/es/cli/)
- Guía de inicio: [docs.aws.amazon.com/cli/latest/userguide/cli-chap-getting-started.html](https://docs.aws.amazon.com/cli/latest/userguide/cli-chap-getting-started.html)
:::

## Instalación

La AWS CLI v2 está disponible para Windows, macOS y Linux. La guía oficial de instalación cubre
los tres sistemas operativos con instaladores nativos:

::: info Guía de instalación oficial
[docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html](https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html)
:::

Una vez instalada, verifica que quedó correctamente disponible en tu terminal:

```bash
aws --version
```

![Resultado de aws --version](/assets/01-fundaments/05-cli/cli-version.png)

El resultado muestra la versión de la CLI, la versión de Python que usa internamente, el sistema
operativo y la arquitectura del procesador. En este caso: CLI v2.34.58 sobre Python 3.14.5 en
Windows 11 con arquitectura AMD64.

## Configuración

Antes de poder ejecutar cualquier comando contra AWS necesitas autenticarte — indicarle a la CLI
con qué cuenta y con qué permisos debe operar.

```bash
aws configure
```

Este comando interactivo solicita cuatro datos:

```
AWS Access Key ID [None]:       → tu clave de acceso
AWS Secret Access Key [None]:   → tu clave secreta
Default region name [None]:     → región por defecto (ej: us-east-1)
Default output format [None]:   → formato de salida (json, table, text)
```

Estos datos se guardan en `~/.aws/credentials` (Linux/macOS) o `%USERPROFILE%\.aws\credentials`
(Windows) y son leídos automáticamente por la CLI en cada comando.

### Cómo Obtener las Access Keys

La forma de obtener las credenciales depende de cómo accedas a AWS. Si usas una cuenta
corporativa o de laboratorio con **AWS IAM Identity Center** (SSO), las credenciales se
obtienen desde el portal de acceso:

![Portal de acceso de AWS](/assets/01-fundaments/05-cli/cli-portal-acceso.png)

En el portal de acceso verás las cuentas a las que tienes acceso. Al expandir una cuenta y
hacer clic en **"Claves de acceso"** (resaltado en rojo en la imagen), se abre un panel con
tres opciones para configurar la autenticación:

![Opciones de credenciales](/assets/01-fundaments/05-cli/cli-credenciales.png)

**Opción recomendada — AWS IAM Identity Center (SSO):** Configura la CLI para obtener
credenciales automáticamente a través de SSO. Ejecutas `aws configure sso` una sola vez,
introduces la URL del portal y la región, y desde entonces la CLI renueva las credenciales
sola sin que tengas que copiar y pegar nada manualmente.

```bash
aws configure sso
# SSO session name: mi-sesion
# SSO start URL: https://identitycenter.amazonaws.com/ssoins-...
# SSO region: eu-west-1
# → Abre el navegador para autenticarte una vez
```

**Opción 1 — Variables de entorno:** Copias y pegas las tres variables directamente en tu
terminal. Son temporales — solo duran mientras esa sesión de terminal esté abierta.

```bash
# Windows (CMD / PowerShell)
SET AWS_ACCESS_KEY_ID=ASIA...
SET AWS_SECRET_ACCESS_KEY=...
SET AWS_SESSION_TOKEN=...

# Linux / macOS
export AWS_ACCESS_KEY_ID=ASIA...
export AWS_SECRET_ACCESS_KEY=...
export AWS_SESSION_TOKEN=...
```

**Opción 2 — Archivo de credenciales:** Agrega un perfil con nombre al archivo
`~/.aws/credentials`. Permite tener múltiples cuentas configuradas y alternar entre ellas
con el flag `--profile`.

```ini
# ~/.aws/credentials
[StudentLabAccess]
aws_access_key_id     = ASIA...
aws_secret_access_key = ...
aws_session_token     = ...
```

```bash
# Usar un perfil específico en cualquier comando
aws s3 ls --profile StudentLabAccess
```

**Opción 3 — Valores individuales en el SDK:** Para uso directo en código cuando construyes
aplicaciones con los SDKs de AWS (boto3 en Python, AWS SDK para JavaScript, etc.).

::: warning Las credenciales de Sandbox son temporales
En cuentas de laboratorio o Sandbox, las credenciales **tienen una duración limitada** (generalmente
entre 1 y 4 horas) y se renuevan cada vez que reinicias el lab. Si un comando empieza a devolver
errores de autenticación, vuelve al portal, obtén nuevas credenciales y repite la configuración.
Esto no ocurre en cuentas reales con usuarios IAM permanentes o con SSO bien configurado.
:::

### Verificar la Identidad Activa

Antes de ejecutar cualquier comando, es buena práctica confirmar con qué identidad estás operando:

```bash
aws sts get-caller-identity
```

```json
{
    "UserId": "AROA...:session-name",
    "Account": "123456789012",
    "Arn": "arn:aws:sts::123456789012:assumed-role/RoleName/session"
}
```

Este comando llama al **STS** (Security Token Service, servicio de tokens de seguridad) y devuelve
el ID de usuario, el número de cuenta y el ARN del rol o usuario activo. Si devuelve error, las
credenciales no están configuradas o han expirado.

## Estructura de los Comandos

Toda la CLI de AWS sigue una estructura consistente y predecible:

```bash
aws [opciones-globales] <servicio> <acción> [parámetros]
```

```
aws          → el binario base
s3           → el servicio (cada servicio tiene su propio subcomando)
ls           → la acción sobre ese servicio
s3://bucket  → parámetros de la acción
```

Las **opciones globales** se escriben antes del servicio y aplican a cualquier comando:

| Opción global | Descripción |
|---|---|
| `--profile <nombre>` | Usa un perfil de credenciales específico |
| `--region <región>` | Sobrescribe la región configurada por defecto |
| `--output <formato>` | Formato de salida: `json`, `table`, `text`, `yaml` |
| `--query <expresión>` | Filtra la salida con JMESPath |
| `--no-cli-pager` | Evita que la salida se pagine (útil en scripts) |

Los **servicios** tienen su propio subcomando que generalmente coincide con el nombre corto del
servicio en AWS. Algunos ejemplos frecuentes:

| Servicio AWS | Subcomando CLI | Ejemplo de uso |
|---|---|---|
| Amazon S3 | `s3` / `s3api` | `aws s3 ls` |
| Amazon EC2 | `ec2` | `aws ec2 describe-instances` |
| AWS IAM | `iam` | `aws iam list-users` |
| AWS Lambda | `lambda` | `aws lambda list-functions` |
| Amazon RDS | `rds` | `aws rds describe-db-instances` |
| AWS STS | `sts` | `aws sts get-caller-identity` |
| Amazon ECS | `ecs` | `aws ecs list-clusters` |
| AWS CloudFormation | `cloudformation` | `aws cloudformation list-stacks` |

::: info Referencia completa de comandos por servicio
[docs.aws.amazon.com/cli/latest/reference](https://docs.aws.amazon.com/cli/latest/reference/)
:::

### El Comando `help`

Cada nivel de la CLI tiene su propio `help`. Es la forma más rápida de descubrir qué hace un
comando y qué parámetros acepta, sin salir de la terminal:

```bash
aws help                  # ayuda general
aws s3 help               # todos los comandos de S3
aws s3 cp help            # parámetros específicos de s3 cp
aws ec2 describe-instances help
```

## Operaciones Generales: Listar, Crear y Eliminar

La CLI sigue convenciones de nomenclatura consistentes entre servicios. Una vez que aprendes
el patrón de un servicio, puedes intuir el de los demás.

### Listar recursos

Los comandos de listado generalmente empiezan por `list-` o `describe-`:

```bash
# S3 — listar buckets
aws s3 ls

# S3 — listar objetos dentro de un bucket
aws s3 ls s3://nombre-del-bucket/
aws s3 ls s3://nombre-del-bucket/ --recursive   # incluye subdirectorios

# EC2 — listar instancias
aws ec2 describe-instances

# EC2 — listar solo las instancias en ejecución
aws ec2 describe-instances \
  --filters "Name=instance-state-name,Values=running"

# IAM — listar usuarios
aws iam list-users

# Lambda — listar funciones
aws lambda list-functions

# ECS — listar clusters
aws ecs list-clusters
```

### Crear recursos

Los comandos de creación usan `create-`, `run-` o `make-` según el servicio:

```bash
# S3 — crear un bucket (mb = make bucket)
aws s3 mb s3://mi-primer-bucket
aws s3 mb s3://mi-bucket-eu --region eu-west-1

# S3 — subir un archivo
aws s3 cp archivo.txt s3://mi-bucket/
aws s3 cp s3://mi-bucket/archivo.txt ./local/   # dirección inversa

# S3 — sincronizar una carpeta completa
aws s3 sync ./mi-carpeta/ s3://mi-bucket/carpeta/

# EC2 — lanzar una instancia
aws ec2 run-instances \
  --image-id ami-0c02fb55956c7d316 \
  --instance-type t3.micro \
  --key-name mi-llave \
  --region us-east-1

# IAM — crear un usuario
aws iam create-user --user-name juan-dev

# Lambda — crear una función (requiere un zip con el código)
aws lambda create-function \
  --function-name mi-funcion \
  --runtime python3.12 \
  --role arn:aws:iam::123456789012:role/mi-rol \
  --handler lambda_function.handler \
  --zip-file fileb://funcion.zip
```

### Eliminar recursos

Los comandos de borrado usan `delete-`, `terminate-` o `remove-`:

```bash
# S3 — eliminar un objeto
aws s3 rm s3://mi-bucket/archivo.txt

# S3 — eliminar un bucket vacío
aws s3 rb s3://mi-bucket

# S3 — eliminar un bucket con todo su contenido
aws s3 rb s3://mi-bucket --force

# EC2 — detener una instancia (apagar, no eliminar)
aws ec2 stop-instances --instance-ids i-0abc1234567890

# EC2 — terminar (eliminar) una instancia
aws ec2 terminate-instances --instance-ids i-0abc1234567890

# IAM — eliminar un usuario
aws iam delete-user --user-name juan-dev

# Lambda — eliminar una función
aws lambda delete-function --function-name mi-funcion
```

::: danger Cuidado con `--force` y `terminate`
`terminate-instances` en EC2 es **irreversible** — la instancia y su disco raíz se eliminan
permanentemente. Lo mismo con `rb --force` en S3: elimina el bucket y **todos** sus objetos
sin confirmación. Siempre verifica dos veces antes de ejecutar comandos destructivos,
especialmente en entornos de producción.
:::

## Formatos de Salida

Por defecto la CLI devuelve JSON, pero puedes cambiar el formato con `--output`:

### JSON (por defecto)
Ideal para procesar la salida con scripts o herramientas como `jq`:

```bash
aws ec2 describe-instances --output json
```

```json
{
    "Reservations": [
        {
            "Instances": [
                {
                    "InstanceId": "i-0abc1234567890",
                    "InstanceType": "t3.micro",
                    "State": { "Name": "running" }
                }
            ]
        }
    ]
}
```

### Table
Formatea la salida como una tabla legible para humanos. Útil para revisar datos en la terminal:

```bash
aws ec2 describe-instances --output table
```

```
----------------------------------------------------------
|                   DescribeInstances                    |
+-------------------+-------------+--------------------+
|    InstanceId     | InstanceType|      State         |
+-------------------+-------------+--------------------+
|  i-0abc1234567890 |  t3.micro   |     running        |
+-------------------+-------------+--------------------+
```

### Text
Salida plana separada por tabulaciones. Útil para procesamiento con `awk`, `cut` o `grep`:

```bash
aws iam list-users --output text
```

### YAML
Disponible desde la CLI v2. Más legible que JSON para archivos de configuración:

```bash
aws ec2 describe-instances --output yaml
```

## Filtrar la Salida con `--query`

La opción `--query` usa el lenguaje **JMESPath** para extraer exactamente los campos que necesitas
de respuestas JSON grandes, sin necesidad de herramientas externas:

```bash
# Extraer solo el InstanceId y el estado de todas las instancias
aws ec2 describe-instances \
  --query "Reservations[*].Instances[*].[InstanceId,State.Name]" \
  --output table

# Obtener solo los nombres de los buckets de S3
aws s3api list-buckets \
  --query "Buckets[*].Name" \
  --output text

# Obtener el ID de una instancia filtrando por nombre
aws ec2 describe-instances \
  --filters "Name=tag:Name,Values=mi-servidor" \
  --query "Reservations[0].Instances[0].InstanceId" \
  --output text
```

::: tip JMESPath
La sintaxis `[*]` itera sobre todos los elementos de una lista. El `.` navega por propiedades
anidadas. `[]` aplana listas de listas. Para experimentar con expresiones JMESPath antes de
usarlas en comandos reales: [jmespath.org](https://jmespath.org/terminal.html)
:::

## Exportar la Salida a Archivos

El operador `>` de la terminal redirige la salida de cualquier comando hacia un archivo en lugar
de mostrarla en pantalla. No es específico de AWS — es un operador estándar de los shells Unix
y Windows.

```bash
# Guardar el inventario de instancias EC2 en un JSON
aws ec2 describe-instances --output json > instancias.json

# Guardar la lista de buckets en texto plano
aws s3 ls > mis-buckets.txt

# Guardar en formato tabla para un reporte legible
aws iam list-users --output table > usuarios.txt

# Guardar las funciones Lambda en YAML
aws lambda list-functions --output yaml > funciones.yaml
```

Con `>>` (doble mayor-que) **añades** al final de un archivo existente en lugar de sobreescribirlo:

```bash
# Añadir logs de múltiples regiones en un mismo archivo
aws ec2 describe-instances --region us-east-1 --output text >> inventario.txt
aws ec2 describe-instances --region eu-west-1 --output text >> inventario.txt
```

### Combinar con `--query` para exportaciones precisas

La combinación de `--query` + `--output text` + `>` es especialmente potente para generar
reportes o alimentar otros sistemas:

```bash
# Exportar solo los IDs y estados de instancias a CSV manual
aws ec2 describe-instances \
  --query "Reservations[*].Instances[*].[InstanceId,InstanceType,State.Name]" \
  --output text > instancias.csv

# Exportar nombres de todos los buckets para auditoria
aws s3api list-buckets \
  --query "Buckets[*].Name" \
  --output text > buckets.txt
```

## Comandos Útiles de Diagnóstico

```bash
# Ver la configuración actual (perfil, región, formato)
aws configure list

# Ver todos los perfiles configurados
aws configure list-profiles

# Ver las credenciales del perfil activo (sin mostrar los valores secretos)
aws configure list --profile mi-perfil

# Verificar conectividad y credenciales
aws sts get-caller-identity

# Ver la versión instalada
aws --version
```
